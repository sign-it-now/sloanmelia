Phase 01 of 05

Domain, Hosting & Public Profile Page

Estimated time: 1‚Äì2 sessions ¬∑ Result: sloanmelia.com is live

By the end of this phase, Sloan has a real, publicly accessible URL she can hand to anyone. The profile page we already built gets deployed here.

Step 1.1

Register sloanmelia.com at Namecheap

  1. Go to namecheap.com and search for sloanmelia.com
  2. Add to cart ‚Äî it should be ~$12 for the first year
  3. At checkout, turn OFF auto-renew for now (you can turn it back on later), turn OFF private registration (it's a fee, and a public profile site doesn't need it)
  4. Complete purchase ‚Äî you now own the domain
  5. Go to your Namecheap Dashboard ‚Üí Manage ‚Üí Advanced DNS ‚Äî leave this tab open, you'll come back to it in Step 1.5
sloanmelia.com is the goal. If it's taken (unlikely), try sloan-melia.com or sloanmsmith.com. Do not use a .net or .org ‚Äî .com is the only one that looks professional on a college application.
Step 1.2

Set Up Your GitHub Repository

  1. Log into github.com
  2. Click the green New button to create a new repository
  3. Name it: sloanmelia-site
  4. Set it to Private (nobody needs to read her source code)
  5. Check "Add a README file"
  6. Click Create repository
  7. Click Add file ‚Üí Upload files
  8. Upload the sloan-profile.html file we built ‚Äî rename it index.html before uploading
  9. Commit the file ‚Äî click Commit changes
Rename the file to index.html before uploading. That's the standard name web servers look for as the homepage. If it stays named sloan-profile.html, the URL would be sloanmelia.com/sloan-profile.html instead of just sloanmelia.com.
Step 1.3

Deploy to Netlify

  1. Log into netlify.com (with your GitHub account)
  2. Click Add new site ‚Üí Import an existing project
  3. Choose GitHub as the source
  4. Find and select sloanmelia-site
  5. Leave all settings as-is ‚Äî click Deploy site
  6. Netlify will give you a random URL like quirky-panda-abc123.netlify.app ‚Äî this is your staging URL, the site is already live here
Visit the Netlify URL right now to confirm the site loads correctly before doing anything else. If it shows a blank page, the file name issue from Step 1.2 is the likely cause ‚Äî make sure the file is named index.html.
Step 1.4

Connect Your Custom Domain

  1. In Netlify, go to Site settings ‚Üí Domain management ‚Üí Add custom domain
  2. Type sloanmelia.com and click Verify
  3. Netlify will show you DNS records to add ‚Äî specifically two values: a CNAME record and an A record
  4. Go back to your Namecheap tab (Advanced DNS) and add those records exactly as Netlify shows them
  5. DNS propagation takes 5‚Äì30 minutes ‚Äî go get coffee
  6. Come back and Netlify will show a green checkmark and enable HTTPS automatically (free SSL certificate)
After DNS propagates, sloanmelia.com will serve her profile page over HTTPS. The little padlock icon in browsers means it's secure. Netlify handles this automatically ‚Äî you don't need to do anything for it.
Step 1.5

Set Up Professional Email ‚Äî sloan@sloanmelia.com

  1. Go to zoho.com/mail and click Sign Up For Free
  2. Choose the Forever Free plan (1 user, 5GB)
  3. Enter sloanmelia.com as the domain
  4. Zoho will give you MX records to add to Namecheap DNS ‚Äî add them the same way you added the Netlify records

-- Run this in Supabase SQL Editor (Dashboard ‚Üí SQL Editor ‚Üí New query) CREATE TABLE documents ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE, name text NOT NULL, category text, -- 'transcript', 'recommendation', 'resume', 'certificate', 'other' file_path text, file_size bigint, created_at timestamp WITH TIME ZONE DEFAULT now() ); -- Enable Row Level Security (users can only see their own docs) ALTER TABLE documents ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users see own documents" ON documents FOR ALL USING (auth.uid() = user_id);

Table 2: achievements

CREATE TABLE achievements ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE, title text NOT NULL, category text, -- 'organization', 'volunteer', 'award', 'internship', 'other' description text, start_date date, end_date date, hours integer, role text, -- their position/role if applicable impact text, -- one sentence impact statement created_at timestamp WITH TIME ZONE DEFAULT now() ); ALTER TABLE achievements ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users see own achievements" ON achievements FOR ALL USING (auth.uid() = user_id);

Table 3: college_applications

CREATE TABLE college_applications ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE, school_name text NOT NULL, deadline date, status text DEFAULT 'researching', -- researching, in-progress, submitted, accepted, waitlisted, rejected notes text, portal_url text, created_at timestamp WITH TIME ZONE DEFAULT now() ); ALTER TABLE college_applications ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users see own applications" ON college_applications FOR ALL USING (auth.uid() = user_id);

Step 3.2

Create a Supabase Edge Function (Keeps API Key Safe)

An Edge Function is server-side code that runs on Supabase's servers. The Claude API key lives here ‚Äî the browser never sees it.

  1. Install Supabase CLI: npm install -g supabase
  2. In your terminal, run: supabase login
  3. Create the function: supabase functions new coach
  4. Edit the function file at supabase/functions/coach/index.ts with this code:
import { serve } from "https://deno.land/std@0.168.0/http/server.ts" const ANTHROPIC_API_KEY = Deno.env.get('ANTHROPIC_API_KEY') serve(async (req) => { // Allow CORS from your site const headers = { 'Access-Control-Allow-Origin': 'https://sloanmelia.com', 'Access-Control-Allow-Headers': 'authorization, content-type', 'Content-Type': 'application/json' } if (req.method === 'OPTIONS') return new Response('ok', { headers }) const { messages, system, vaultContext } = await req.json() // Build system prompt with Sloan's data from vault const systemPrompt = `You are Sloan's personal college application coach. You are embedded in her private dashboard at sloanmelia.com. You know everything about her from her Document Vault. SLOAN'S PROFILE: - Full Name: Sloan Smith - School: Alton High School, Alton, Illinois - Graduating: May 2026 - GPA: 3.684 - Target Degree: PhD in Forensic Psychology - Email: sloan@sloanmelia.com VAULT CONTEXT (what she has uploaded): ${vaultContext || 'No documents uploaded yet.'} Your capabilities: 1. Analyze college application forms (paste the form text and you fill it out using her real data) 2. Generate copy-paste ready answers for any application field 3. Answer questions about her study materials (forensic psychology, DSM-5, etc.) 4. Help with college essays and personal statements 5. Explain financial aid concepts (FAFSA, 529 plans, scholarships) 6. Flag what documents are missing for specific applications Always be direct, specific, and use her actual data. Never make up facts about her. When filling forms, format output as: FIELD NAME: [answer] on separate lines.` const response = await fetch('https://api.anthropic.com/v1/messages', { method: 'POST', headers: { 'x-api-key': ANTHROPIC_API_KEY, 'anthropic-version': '2023-06-01', 'content-type': 'application/json' }, body: JSON.stringify({ model: 'claude-sonnet-4-6', max_tokens: 2048, system: systemPrompt, messages: messages }) }) const data = await response.json() return new Response(JSON.stringify(data), { headers }) })
  1. Deploy the function: supabase functions deploy coach --no-verify-jwt
  2. Add the secret API key: supabase secrets set ANTHROPIC_API_KEY=sk-ant-YOUR-KEY-HERE
The Edge Function is now live at: https://YOUR-PROJECT-REF.supabase.co/functions/v1/coach ‚Äî the frontend calls THIS URL, not Anthropic directly. The API key never leaves the server.

The Coach System ‚Äî Full Implementation Guide

The Coach is the most important feature. Here's exactly how it works end-to-end so you can troubleshoot, extend, and explain it.

How It Works

The Data Flow ‚Äî What Happens When Sloan Types a Message

  1. Sloan types a message in the Coach chat tab
  2. The frontend queries Supabase to get her documents list and achievements list
  3. Both are formatted into a "vault context" string ‚Äî something like "Uploaded documents: High School Transcript (uploaded Jan 2026), Recommendation Letter from Kelly Paige (uploaded Jan 2026)..."
  4. The frontend sends a POST request to the Supabase Edge Function with: the message, conversation history, and vault context
  5. The Edge Function builds a system prompt that includes Sloan's full profile + vault context, then calls the Claude API
  6. Claude responds with full knowledge of who she is and what she has
  7. Response streams back to the chat UI
Coach Context

What the Coach Already Knows (Hardcoded in the System Prompt)

The Coach system prompt includes all of this by default ‚Äî no uploads needed: PERSONAL: - Full name: Sloan Smith - School: Alton High School, Alton, IL - Graduating: May 2026 - GPA: 3.684 - Email: sloan@sloanmelia.com - Goal: PhD in Forensic Psychology SENIOR YEAR COURSES: AP Psychology, AP Statistics, College Prep English 4 (H) AP, Human Body Systems PLTW 2, Biomedical Internship, Sociology, Environmental Science, Spanish 2A & 2B ACHIEVEMENTS (pre-loaded): - Patient Enrichment Program (Founder ‚Äî ongoing annual program) - Mu Alpha Theta (National Math Honor Society) - Link Crew (Peer Mentor) - Salvation Army (Volunteer) - Church Nursery (Volunteer) - Math Tutoring (Peer Tutor) FORENSIC PSYCH KNOWLEDGE: The Coach also has access to all 16 study posters built in this conversation ‚Äî it knows the DSM-5 reference, theorists, research methods, ethics code, and supporting courses.
